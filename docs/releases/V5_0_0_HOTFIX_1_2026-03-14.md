# Deep-Sea Nexus v5.0.0 Hotfix 1

Date: 2026-03-14

## Release Position

- Base release: `v5.0.0` official
- Patch type: runtime hardening + security fix + scope-isolation fix
- Status: patch ready for production rollout on top of `v5.0.0`

## What Was Fixed

### 1) Security: frontmatter parser no longer executes code

- File: `scripts/import_sessions.py`
- Change:
  - replaced unsafe `eval(...)` parsing path with `yaml.safe_load(...)`
  - only accepts parsed `list/dict` values for structured frontmatter fields
- Risk removed:
  - arbitrary code execution via crafted session frontmatter input

### 2) Memory v5: scoped path hardening

- File: `memory_v5/layout.py`
- Change:
  - hardened scope segment sanitizer against path traversal / absolute path style input
  - normalized separators and stripped unsafe `.` / `..` segments
- Risk removed:
  - writes escaping configured memory root through malicious `MemoryScope` values

### 3) Memory v5: category records are now scope-key isolated

- Files:
  - `memory_v5/service.py`
  - `memory_v5/layout.py`
- Change:
  - category record ID now includes scope-key digest (`agent/user/app/run/workspace`)
  - category markdown filename also receives scope digest when app/run/workspace qualifiers exist
- Risk removed:
  - cross-scope category overwrite / summary pollution for same agent+user lanes

### 4) Runtime stability: sync/async event handling hardening

- Files:
  - `core/event_bus.py`
  - `plugins/session_manager.py`
- Change:
  - event bus construction no longer depends on pre-existing asyncio default loop
  - event dispatch now safely handles both async callbacks and sync callbacks that return awaitables
  - session manager now uses a sync/async-safe coroutine launcher in all sync-facing methods
- Risk removed:
  - `RuntimeError: no running event loop` in sync paths after runtime loop churn

### 5) Operational consistency: restored current hook wrapper entrypoints

- Files:
  - `hooks/agent_end/run_save.sh`
  - `hooks/before_agent_start/run_recall.sh`
  - `.gitignore`
- Change:
  - restored wrapper scripts expected by current tests and runtime docs
  - adjusted `.gitignore` to keep runtime hook wrappers versioned
- Impact:
  - shell-entrypoint validation and runtime hook chain remain aligned
