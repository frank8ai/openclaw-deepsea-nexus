# OpenClaw Deep-Sea Nexus v5.3.0 Release

Date: 2026-03-21

## Release Decision

- Version: `v5.3.0`
- Status: feature release
- Base: `v5.2.0` feature baseline
- Scope: execution_guard, execution-governor bridge, and report-first tool safety

## Release Summary

`v5.3.0` extends the `v5.2.0` runtime middleware lane with a second internal layer: `execution_guard`.

This release does not replace the existing memory pipeline. It adds a report-first safety policy layer between tool execution context and execution-governor observability.

Included in `v5.3.0`:

- internal `execution_guard` plugin
- report-first `allow / ask / block / context` decisions
- secrets / credentials / workspace-boundary / second-brain asset checks
- shell-obfuscation and network-bootstrap risk detection
- `tool_event.metadata.guard` persistence in Memory v5
- execution-governor guardrails export:
  - `scripts/sync_execution_governor_guardrails.py`
- execution guard metrics reporting:
  - `scripts/execution_guard_report.py`
- health/path visibility for:
  - `execution_guard`
  - guardrails output path
  - execution guard metrics path

## Why This Release Exists

`v5.2.0` established runtime middleware for tool-output normalization and token-aware capture.

`v5.3.0` adds a complementary question:

- not only “what should we remember from tool output?”
- but also “how risky was the tool action itself?”

This release borrows the core idea from `nah`-style PreToolUse safety systems, but keeps Deep-Sea Nexus as the policy source of truth.

## Release Contract

`v5.3.0` keeps the stable contract from earlier v5 releases:

- sync compatibility API (`nexus_init / nexus_recall / nexus_add`)
- async plugin runtime lifecycle (`create_app()`)
- Memory v5 scoped storage (`MemoryScope / MemoryV5Service`)
- context-governed recall / inject / compress / rescue / replay loop
- evidence-gated durable decision writes
- local deployment and verification toolchain

New internal runtime additions in this release:

- `execution_guard` plugin
- execution-governor guardrails export
- structured guard decisions attached to `tool_event`

Current boundary:

- internal in `v5.3.0`
- report-only by default
- no new public Python API
- OpenClaw adapter remains minimal in this release
