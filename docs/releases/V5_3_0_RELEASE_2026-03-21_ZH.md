# OpenClaw Deep-Sea Nexus v5.3.0 发布说明

日期：2026-03-21

## 发布决定

- 版本：`v5.3.0`
- 状态：feature release
- 基线：`v5.2.0` feature baseline
- 范围：`execution_guard`、execution-governor bridge，以及 report-first 工具安全治理

## 发布摘要

`v5.3.0` 在 `v5.2.0` 的 runtime middleware 基础上，新增了第二层内部 runtime 能力：`execution_guard`。

这个版本并不替换现有记忆链路，而是在工具执行上下文与 execution-governor 联动面之间增加一层 report-first 安全判定：

- 识别 secrets / credentials 风险
- 识别工作区越界访问与写删风险
- 识别第二大脑关键资产访问风险
- 识别 `curl|bash` / `wget|sh` / `powershell -enc` / decode+exec 这类高风险 shell 模式
- 生成 `allow / ask / block / context` 决策

本次 `v5.3.0` 包含：

- 新增内部 `execution_guard` 插件
- report-first `allow / ask / block / context` 风险决策
- `tool_event.metadata.guard` 风险结果持久化
- execution-governor guardrails 导出脚本：
  - `scripts/sync_execution_governor_guardrails.py`
- execution guard metrics 报告脚本：
  - `scripts/execution_guard_report.py`
- `health --json` 和 `paths --json` 中新增：
  - `execution_guard` 插件摘要
  - `execution_guard_metrics_path`
  - `execution_governor_guardrails_path`
  - `execution_guard_last_metrics`

## 为什么有这个版本

`v5.2.0` 解决的是：

- 工具输出如何压缩
- 哪些内容值得进入 capture
- token 成本如何估算

`v5.3.0` 进一步补的是：

- 工具动作本身是否安全
- 风险应该如何分类
- 宿主在不破坏现有工作流的前提下，如何先接入安全 recommendation

它吸收了 `nah` 这类 PreToolUse 安全系统的核心思想，但仍由 `Deep-Sea Nexus` 维护策略真源。

## 发布契约

`v5.3.0` 保持此前 v5 系列稳定契约不变：

- sync compatibility API（`nexus_init / nexus_recall / nexus_add`）
- async plugin runtime lifecycle（`create_app()`）
- Memory v5 scoped storage（`MemoryScope / MemoryV5Service`）
- context-governed recall / inject / compress / rescue / replay loop
- evidence-gated durable decision writes
- 本地 deploy / doctor / smoke / benchmark 工具链

本次新增的内部能力：

- `execution_guard` 插件
- execution-governor guardrails 单一真源导出
- `tool_event` 上的 guard metadata

当前边界：

- 仍是 internal 能力，不是 public Python API
- 默认 `report_only`
- 当前只交付最小 OpenClaw 适配入口
- 默认不对宿主做硬阻断
