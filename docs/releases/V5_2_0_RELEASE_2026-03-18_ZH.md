# OpenClaw Deep-Sea Nexus v5.2.0 发布说明

日期：2026-03-18

## 发布决定

- 版本：`v5.2.0`
- 状态：feature release
- 基线：`v5.0.1` patch baseline
- 范围：runtime middleware、Windows / shell 兼容性修复，以及当前 release-pack 文档对齐

## 发布摘要

`v5.2.0` 引入了一层新的内部 runtime 中间层，用来在工具输出进入记忆链路前先做压缩、结构化和 token 估算；同时完成了一轮跨平台清理，使当前 release pack 在 Windows 上更可用、更容易运维。

本次 `v5.2.0` 包含：

- 新增内部 `runtime_middleware` 插件
- 在 capture 前执行 RTK 风格工具输出压缩
- 增加 token 计量：
  - `token_before`
  - `token_after`
  - `reduction_ratio`
  - `salience_score`
- 新增 token-aware capture gate，用于决定：
  - store
  - skip
  - aggregate
- `Memory v5` 新增 `tool_event` 结构化写入
- 新增 OpenClaw-first `tool-call` 适配入口：
  - `hooks/tool-call/runtime_middleware_capture.py`
- 在 CLI 状态里暴露 runtime middleware 健康信息与路径
- 新增 middleware metrics 报告脚本：
  - `scripts/runtime_middleware_report.py`
- Windows 运维兼容性增强：
  - PowerShell / CMD doctor 入口
  - `bash.cmd` shell-wrapper 支持
  - isolated 环境下更安全的 home/path 解析
  - 当前 runtime 所需 hook wrapper 正式纳入仓库

## 为什么有这个版本

`v5.0.0` 确立了当前架构、API 和治理链路。

`v5.0.1` 收紧了 lifecycle operator 的输出与巡检能力。

`v5.2.0` 则把 runtime 向前推进了一步：

- 不只是“记忆如何被存储和召回”
- 还包括“高噪声工具输出如何在进入记忆之前被治理”

本次发版在 API 范围上保持克制：

- 不新增 public Python API
- 不绕过现有 evidence gate
- 不替换当前 recall / inject 排序主逻辑

## 发布契约

`v5.2.0` 保持此前 v5 系列的稳定契约：

- sync compatibility API（`nexus_init / nexus_recall / nexus_add`）
- async plugin runtime lifecycle（`create_app()`）
- Memory v5 scoped storage（`MemoryScope / MemoryV5Service`）
- context-governed recall / inject / compress / rescue / replay loop
- evidence-gated durable decision writes
- 显式 lifecycle governance 与本地验证工具链

本次新增的内部 runtime 能力：

- `runtime_middleware` 插件
- `tool_event` scoped durable items
- middleware metrics 与 evidence snapshots

当前边界：

- 在 `v5.2.0` 中仍然是 internal
- 可通过 `health / paths / report scripts` 观测
- 尚未作为面向外部 runtime 的稳定 public integration API

