# OpenClaw Deep-Sea Nexus v5.0.1 Patch Release

Date: 2026-03-14

## Release Decision

- Version: `v5.0.1`
- Status: patch release
- Base: `v5.0.0` official release baseline
- Scope: lifecycle operator and release-pack documentation tightening

## Patch Summary

This patch does not change the main product shape. It makes the current
lifecycle governance path more operable and easier to audit.

Included in `v5.0.1`:

- thresholded lifecycle status for maintenance runs
- flattened `alerts[]` output for risky scopes
- sorted `hot_scopes[]` summaries for operators
- explicit `recommendations[]` in lifecycle reports
- configurable lifecycle alert thresholds via `memory_v5.lifecycle_alerts.*`
- `--no-alerts` mode to keep report-first operation explicit
- current release-pack docs aligned to the patch baseline

## Why This Patch Exists

`v5.0.0` established the official current architecture, API surface, and Memory
v5 lifecycle governance primitives.

`v5.0.1` tightens the last operator-facing step:

- less raw report reading
- clearer scope-level risk posture
- more consistent handoff and maintenance output
- no silent runtime auto-mutation added

## Release Contract

`v5.0.1` keeps the same stable contract as `v5.0.0`:

- sync compatibility API (`nexus_init / nexus_recall / nexus_add`)
- async plugin runtime lifecycle (`create_app()`)
- Memory v5 scoped storage (`MemoryScope / MemoryV5Service`)
- context-governed recall / inject / compress / rescue / replay loop
- evidence-gated durable decision writes
- explicit lifecycle governance (`audit_lifecycle`, explicit archive, explicit backfill)
- local deployment and verification toolchain (`deploy / doctor / smoke / benchmark`)

Patch-specific operator additions:

- lifecycle maintenance now emits `status.level`
- lifecycle maintenance now emits `alerts[]`
- lifecycle maintenance now emits `hot_scopes[]`
- lifecycle maintenance now emits `recommendations[]`
