# OpenClaw Deep-Sea Nexus v5.2.0 Release

Date: 2026-03-18

## Release Decision

- Version: `v5.2.0`
- Status: feature release
- Base: `v5.0.1` patch baseline
- Scope: runtime middleware, Windows/shell compatibility, and release-pack documentation alignment

## Release Summary

`v5.2.0` adds a new internal runtime layer for tool-output capture and folds in a cross-platform cleanup pass that makes the current release pack more operable on Windows.

Included in `v5.2.0`:

- internal `runtime_middleware` plugin
- RTK-style tool-output compression before capture
- token accounting with:
  - `token_before`
  - `token_after`
  - `reduction_ratio`
  - `salience_score`
- token-aware capture gating for:
  - store
  - skip
  - aggregate
- `tool_event` writes into `Memory v5`
- OpenClaw-first `tool-call` adapter:
  - `hooks/tool-call/runtime_middleware_capture.py`
- runtime middleware health and path visibility in CLI status
- middleware metrics reporting:
  - `scripts/runtime_middleware_report.py`
- Windows operational compatibility improvements:
  - PowerShell / CMD doctor entrypoints
  - shell-wrapper support via `bash.cmd`
  - safer home/path resolution in isolated environments
  - current hook wrapper scripts added to the tracked runtime surface

## Why This Release Exists

`v5.0.0` established the official current architecture.

`v5.0.1` tightened lifecycle operator output.

`v5.2.0` extends the runtime one step earlier in the pipeline:

- not only how memory is stored and recalled
- but also how noisy tool output is normalized before it becomes memory

This release is intentionally conservative about API scope:

- no new public Python API
- no bypass around existing evidence gates
- no replacement of current recall ranking logic

## Release Contract

`v5.2.0` keeps the stable contract from earlier v5 releases:

- sync compatibility API (`nexus_init / nexus_recall / nexus_add`)
- async plugin runtime lifecycle (`create_app()`)
- Memory v5 scoped storage (`MemoryScope / MemoryV5Service`)
- context-governed recall / inject / compress / rescue / replay loop
- evidence-gated durable decision writes
- explicit lifecycle governance and local verification toolchain

New internal runtime additions in this release:

- `runtime_middleware` plugin
- `tool_event` scoped durable items
- middleware metrics and evidence snapshots

Current boundary:

- internal in `v5.2.0`
- observable through health / paths / report scripts
- not yet a public integration API for external runtimes
